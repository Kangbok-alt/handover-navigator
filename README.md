# Handover Navigator 🧭

> 전임자가 인수인계 없이 떠나 자료 더미만 남았을 때, 그 더미를 **맥락 0인 후임자가 첫 30일을 버틸 수 있는 인수인계 문서**로 바꿔주는 Claude 스킬.

[멘토라이브러리 "AI 에이전트를 고용하라" 클래스] 실습 과제로 제작했습니다. 모든 샘플 데이터는 **가상**이며 민감 정보는 포함돼 있지 않습니다.

---

## 무엇을 하나요

자료 폴더(문서·이메일·메모·스프레드시트·회의록)를 입력하면 4종 문서를 만듭니다.

1. **인수인계 요약서** — 1페이지 핵심 + 6블록 상세 (상사 보고용)
2. **미결 과제 트래커** — 과제/상태/담당자/기한/신뢰도/출처 표
3. **첫 30일 온보딩 플랜** — 1주·2주·한 달 체크리스트
4. **물어볼 질문 리스트** — 자료로 확인 불가한 빈칸을 "누구에게 무엇을 물을지"로 정리

## 왜 단순 요약기와 다른가

낯선 사람의 자료 더미는 신뢰할 수 없습니다. 이 스킬의 핵심은 **모르는 것을 지어내지 않는 것**입니다.

- **출처 추적** — 모든 사실에 근거 파일을 표기. 근거 없는 단정은 쓰지 않음.
- **신뢰도 3단계** — `문서확인` / `추론`(근거 동반) / `UNKNOWN`. 완성도와 분리.
- **UNKNOWN → 질문으로** — 빈칸을 추측으로 메우는 대신 확인할 질문으로 전환.
- **상태 표준화** — "홀딩", "거의 끝", "?" 같은 제각각 표현을 5개 표준 어휘로 정규화.
- **데이터 계약** — 원본은 읽기 전용. 산출물은 별도 폴더에만. 원본·민감자료 보호.

## 폴더 구조

```
handover-navigator/
├── SKILL.md                      # 스킬 본체 (라우터 + 원칙 + 워크플로)
├── references/
│   ├── methodology.md            # 6블록 캔버스 상세 추출 기준, 신뢰도 판정
│   └── states.md                 # 진행 상태 표준 어휘 + 별칭 매핑
├── assets/templates/             # 산출물 템플릿
│   ├── handover-summary.md
│   ├── open-tasks-tracker.md
│   ├── onboarding-30day.md
│   ├── questions-to-ask.md
│   └── new-plan-draft.md         # 신규 기획서 초안 (옵션)
├── evals/                        # 검증
│   ├── trigger-eval.json         # 트리거 평가셋 (20문항)
│   ├── eval-report.md            # RED vs GREEN 환각 대조 + 발동 정확도(36/36)
│   ├── red-output-haiku.md       # RED 원본 출력(스킬 없음, 빠른 모델)
│   └── red-output-opus.md        # RED 원본 출력(스킬 없음, 강한 모델)
├── examples/                     # 예시 산출물 (스킬 적용 결과, 5종)
│   ├── handover-summary-EXAMPLE.md
│   ├── open-tasks-tracker-EXAMPLE.md
│   ├── onboarding-30day-EXAMPLE.md
│   ├── questions-to-ask-EXAMPLE.md
│   └── new-plan-draft-EXAMPLE.md
└── sample/                       # 데모용 가상(비민감) 자료 더미
```

검증은 `evals/`에 있습니다. ① **환각 방지**: 같은 샘플을 스킬 없이(RED) vs 스킬 적용(GREEN)으로 대조 — 스킬 없을 때 모델은 없는 매출 목표를 지어내고, 한 달 지난 마감을 "임박"으로, 상태 불명을 "미진행"으로 단정했으나, 스킬은 이를 출처·UNKNOWN·Blocker로 바로잡습니다. ② **발동 정확도**: 트리거/근접오답 혼합 36문항에서 **36/36** 정확.

## 써보기 (데모)

Claude(Cowork/Claude Code 등 스킬 지원 환경)에서 이 스킬을 설치한 뒤:

```
이 sample 폴더가 퇴사한 전임자가 남긴 자료야. 인수인계서 만들어줘.
```

`sample/` 폴더는 일부러 지저분하게 만들어, 스킬이 혼란을 어떻게 정리하고 무엇을 `UNKNOWN`으로 남기는지 보여줍니다. 예를 들어 "심평원 5/29 마감"은 Blocker 리스크로, "○○ 프로젝트 실제 담당자"는 질문 리스트로 빠집니다.

## 6블록 캔버스

| 블록 | 내용 |
| --- | --- |
| 현황 | 진행 중 업무가 지금 어느 단계인가 |
| 미결 | 안 끝난 일, 마감 임박, 대기 중인 결정 |
| 자산 | 핵심 파일·데이터·계정·계약 위치 맵 |
| 관계 | 내부·외부 핵심 인물, 담당, 연락처 |
| 맥락 | 왜 이렇게 했나, 과거 결정 배경 |
| 리스크 & 다음 | 지뢰, 첫 2주 할 일 |
