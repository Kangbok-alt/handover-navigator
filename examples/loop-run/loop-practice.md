# handover-navigator 루프 실습 — Generator·Evaluator + /loop

오전 과제(handover-navigator)의 유일한 약점은 **완료 게이트가 self-check**라는 점이었습니다(자기 평가의 함정).
오후 실습은 그 게이트를 **측정 가능한 Rubric**으로 바꾸고, **작성(Generator)과 채점(Evaluator)을 분리**해
**모든 기준이 PASS될 때까지 반복**하는 루프로 메웁니다. → 하네스 원칙1(역할 분리) + 루프(반복·독립검증).

---

## 0. 준비

- `handover-navigator` 스킬 설치됨
- `sample/` 폴더(가상 전임자 자료) 접근 가능
- 작업 폴더: `./handover-loop/`
  - `output/` — Generator가 만든 인수인계 문서
  - `feedback.md` — Evaluator가 쓴 채점 결과

---

## 1. 합격 기준 (Rubric) — 어제의 완료 게이트를 PASS/FAIL로

기계·채점자가 참·거짓으로 판정할 수 있게 9개로 분해했습니다.

| # | 기준 | PASS 조건 |
| --- | --- | --- |
| 1 | 출처 추적 | 모든 단정적 사실에 `(출처: 파일)` 표기. 무출처 단정 **0건** |
| 2 | 환각 0 | 산출물의 모든 사실이 `sample/` 원본에서 확인됨. 원본에 없는 정보 **0건** |
| 3 | UNKNOWN 처리 | 자료로 확인 불가한 항목은 추측 대신 `UNKNOWN`/질문으로. 추측 채움 **0건** |
| 4 | 상태 정규화 | 모든 상태가 표준 5어휘(미착수/진행중/막힘/완료/불명)로 표기 |
| 5 | 날짜 경과 | 작성일과 비교해 지난 마감(예: 5/22·5/29)은 "경과(미확인)+Blocker"로 |
| 6 | 담당자 | 모든 미결 항목에 담당자 **또는** `UNKNOWN` |
| 7 | 자산 위치 | 모든 자산에 위치(경로·링크) 또는 권한 요청 대상 |
| 8 | 리스크 심각도 | 모든 리스크에 Blocker/Watch/Nice-to-have |
| 9 | 질문 리스트 | 비어 있지 않고, 실제 빈칸(담당·마감·맥락)을 우선순위로 포착 |

> 추가 점검(있으면): 평문 비밀번호/토큰 발견 시 값 대신 `[REDACTED]`로 마스킹.

---

## 2. /loop 한 줄 (클로드 코드용)

```
/loop (비우면 스스로 판단) sample 폴더로 인수인계 문서를 합격선까지 다듬는다

# 멈출 조건 (충족되면 끝낸다)
  · Rubric 9개 항목이 전부 PASS면 멈춘다 (끝 표시 규칙: 전부 PASS만 완료)
  · 또는 3회를 돌려도 PASS 수가 늘지 않으면 멈추고 사람에게 보고한다 (진전 없음 감지)

# 매번 할 일
  · Generator: handover-navigator로 직전 FAIL 항목만 고쳐 다시 생성
  · Evaluator(다른 역할): 원본 sample와 대조해 9개 기준 PASS/FAIL + 근거를 feedback.md에 기록
  · 채점은 작성과 분리한다 (실행자가 자기 합격을 매기지 않음)
  · 카톡 전송 등 되돌릴 수 없는 동작은 사람 승인 후에만
  · 한 바퀴마다 무엇을 고쳤는지 한 줄로 남긴다
```

## 3. 간략 버전 (한 세션, 두 역할 — 빠른 실습용)

> 한계: 한 모델이 두 역할을 흉내 내므로 *진짜* 분리는 아님. 개념을 빠르게 체험할 때 사용.

```
# 이 프롬프트가 하려는 일
sample 폴더(퇴사한 전임자 자료)로 인수인계 문서를 만들고, 합격선까지 자동으로 끌어올린다.
같은 AI에게 만들기와 채점을 모두 맡기면 자기 결과에 후해지므로 두 역할을 나눈다.

# 등장 인물
Generator (제작자)
  · 입력: sample/ 폴더 + (있으면) 직전 회차 feedback.md
  · 출력: 인수인계 문서 (요약서·미결 트래커·30일 온보딩·질문 리스트)
  · 임무: handover-navigator 스킬로 작성. 2회차부터는 FAIL 항목만 수정
  · 안 함: 자기 결과 채점, 합격 기준 변경

Evaluator (채점자)
  · 입력: 방금 만든 인수인계 문서 + 원본 sample/
  · 출력: 9개 기준별 PASS/FAIL + 근거
  · 임무: 원본과 대조해 냉정하게 판정. 점수보다 "무엇이 왜 FAIL인지"를 먼저 적기
  · 안 함: 문서 직접 수정, 합격 기준 변경

# 합격 기준 (1절 Rubric 9개)
1 출처추적 / 2 환각0 / 3 UNKNOWN처리 / 4 상태정규화 / 5 날짜경과 / 6 담당자 / 7 자산위치 / 8 리스크심각도 / 9 질문리스트

# 진행 순서 (최대 3회차)
한 회차 = Generator 출력 → Evaluator 출력.
2회차부터 Generator는 직전 feedback.md에서 FAIL인 항목만 손본다. 나머지는 절대 건드리지 않는다.

# 종료 조건
9개 항목 모두 PASS면 종료. 아니면 다음 회차로. 최대 3회.
3회를 돌려도 PASS 수가 늘지 않으면 멈추고 남은 FAIL을 사람에게 보고.
```

## 4. 상세 버전 (Task 서브에이전트, 파일로만 주고받기 — 진짜 분리)

> 클로드 코드의 Task 도구로 Generator/Evaluator를 별도 서브에이전트로 띄운다.
> 둘은 같은 메시지 창에서 대화하지 않고 **파일로만** 주고받으며, 매 회차 새 컨텍스트로 시작한다.

```
# 작업 디렉토리
./handover-loop/
  ├─ output/        (Generator의 인수인계 문서)
  └─ feedback.md    (Evaluator의 PASS/FAIL + 근거)

# 입력
원본 자료 폴더: ./sample/   (가상 전임자 자료, 읽기 전용)

# 메인 에이전트 절차 (최대 3회차)
회차마다 순서대로 수행한다.

1. Task 도구로 generator-agent 호출
   입력: ./sample/ + (있으면) ./handover-loop/feedback.md
   산출물: ./handover-loop/output/ (요약서·트래커·온보딩·질문 리스트)

2. Task 도구로 evaluator-agent 호출
   입력: ./handover-loop/output/ + 원본 ./sample/
   산출물: ./handover-loop/feedback.md

3. feedback.md를 읽는다.
   9개 기준 전부 PASS면 STATUS: COMPLETED 선언 후 종료.
   아니면 다음 회차로. 단, 3회차에도 PASS 수가 늘지 않으면 멈추고 보고.

# generator-agent 시스템 프롬프트
너는 인수인계 문서를 만드는 Generator다.
- handover-navigator 스킬의 원칙을 따른다: 모든 사실에 출처, 모르면 UNKNOWN, 원본은 읽기 전용.
- ./sample/ 와 (있으면) ./handover-loop/feedback.md 를 읽는다.
- 1절 Rubric 9개를 목표로 ./handover-loop/output/ 에 문서를 쓴다.
- feedback.md가 있으면 FAIL 항목만 수정한다. 나머지는 절대 건드리지 않는다.
- 수정한 부분에 주석을 단다: <!-- 수정(N회차): [기준#] - [변경 내용] -->
- 원본에 없는 사실을 절대 지어내지 않는다. 불확실하면 UNKNOWN + 질문 리스트로.

# evaluator-agent 시스템 프롬프트
너는 인수인계 문서를 채점하는 Evaluator다.
- ./handover-loop/output/ 의 문서를 ./sample/ 원본과 한 줄씩 대조한다.
- 1절 Rubric 9개를 각각 PASS/FAIL로 판정하고 ./handover-loop/feedback.md 에 기록한다.

  | 기준# | 판정(PASS/FAIL) | 근거(원본 대조, 구체적으로) |
  | ----- | --------------- | -------------------------- |

  특히 기준2(환각): 산출물에서 원본에 없는 사실을 발견하면 그 문장을 인용해 FAIL.
  기준1(출처): 출처 없는 단정 문장을 인용해 FAIL.
- 점수보다 "무엇이 왜 FAIL인지, 무엇을 바꿔야 하는지"를 먼저 적는다.
- 문서를 직접 수정하지 않는다. 합격 기준을 바꾸지 않는다.
```

## 5. 멈출 장치 & 사람의 자리 (루프 안전수칙)

- **반복 상한**: 최대 3회. 넘으면 멈추고 남은 FAIL 보고.
- **진전 없음 감지**: 2회 연속 PASS 수가 같으면 같은 자리를 도는 것 → 멈춤.
- **독립 검증**: Generator는 절대 자기 합격을 매기지 않음(Maker-Checker).
- **끝 표시 규칙**: 9개 전부 PASS일 때만 완료. 조기 완료 선언 금지.
- **승인 게이트**: 카톡 전송·파일 공유 등 되돌릴 수 없는 동작은 **사람 승인 후에만**. (handover-navigator는 '카톡엔 1페이지 요약만' 원칙 유지)
- **사람 몫**: "루프가 고쳤다"가 "내가 이해했다"는 아님 — 최종 판단은 사람이.
